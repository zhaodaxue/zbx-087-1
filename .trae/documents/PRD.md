## 1. 产品概述

队形 Cue 编排器是一款面向舞龙队排练的浏览器端工具，用于编排和预览队形切换序列。用户可通过可视化界面创建、编辑和排序队形口令卡（Cue），系统自动校验编排规则并提供实时预览。

- **核心价值**：将舞龙队排练中的口令卡数字化，提供直观的时间轴编排、规则自动校验和动态预览功能
- **目标用户**：舞龙队教练、编排人员
- **技术特点**：纯前端应用，无后端依赖，支持 Docker 静态部署

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 编排者 | 无需注册 | 完整的 Cue 编排、规则校验、预览和导出功能 |

### 2.2 功能模块

1. **Cue 编排器**：Cue 列表管理、增删改操作、拖拽排序
2. **时间轴**：可视化时间轴展示、累计时长计算
3. **规则引擎**：自动校验三条编排规则、违规高亮提示
4. **预览模块**：时间滑块控制、当前队形实时显示
5. **导出功能**：导出 JSON 格式编排数据

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面 | Cue 列表 | 展示所有 Cue 卡片，支持编辑、删除、拖拽排序 |
| 主页面 | 时间轴 | 横向时间轴，按比例显示各 Cue 时长和累计时间 |
| 主页面 | 规则违规面板 | 实时显示所有规则违规项，点击可定位 |
| 主页面 | 预览区域 | 时间滑块 + 队形代号大字展示 |
| 主页面 | 操作工具栏 | 添加 Cue、导出 JSON、重置等操作 |

## 3. 核心流程

用户打开应用 → 查看默认/空编排 → 添加/编辑/删除 Cue → 拖拽调整顺序 → 实时查看规则校验结果 → 拖动时间滑块预览队形 → 导出 JSON 文件

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：中国红 (#C8102E) 搭配金色 (#D4AF37)，体现舞龙传统文化特色
- **辅助色**：深灰背景 (#1a1a1a)，营造专业编排工具氛围
- **按钮风格**：圆角矩形，悬停有微缩放和发光效果
- **字体**：标题使用 serif 字体增强传统感，正文使用 sans-serif 保证可读性
- **布局风格**：卡片式布局，三栏结构（左侧 Cue 列表 + 中间时间轴预览 + 右侧规则面板）
- **图标风格**：线性图标，统一 24px 尺寸

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 主页面 | Cue 列表 | 卡片列表、拖拽手柄、编辑按钮、删除按钮、颜色标识 |
| 主页面 | 时间轴 | 彩色块时间轴、时间刻度、悬停提示 |
| 主页面 | 规则面板 | 违规条目列表、严重程度标识、定位跳转 |
| 主页面 | 预览区 | 大字号队形代号、时间滑块、播放/暂停 |
| 主页面 | 工具栏 | 添加按钮、导出按钮、总时长显示 |

- 桌面端优先，三栏布局

- 移动端：单栏堆叠，支持触控拖拽

## 5. 业务规则

### 5.1 Cue 数据结构

每条 Cue 包含：
- **名称**：字符串，口令卡名称
- **持续秒数**：正整数，单位秒
- **目标队形代号**：字符串，如"一字"、"螺旋"、"S形"
- **切换方式**：枚举值，"齐步" 或 "跑步"
- **是否过渡**：布尔值，是否为过渡 Cue

### 5.2 编排规则

1. **跑步过渡规则**：相邻两条 Cue 若切换方式均为"跑步"，中间须插入至少 1 条过渡 Cue（持续不少于 3 秒）
2. **总时长规则**：全程总时长不得超过 480 秒
3. **队形连续规则**：同一队形代号不能连续出现 3 次

### 5.3 导出格式

导出 JSON 包含：
- Cue 顺序列表
- 每条 Cue 的累计开始时刻和结束时刻
- 总时长
